# คู่มือการใช้งานระบบ RIMS (Revenue Insurance Management System)
> ระบบบริหารจัดการข้อมูลการจัดเก็บรายได้ค่ารักษาพยาบาลและลูกหนี้ค่ารักษาพยาบาล

---

## 1. แนะนำโปรแกรม (Introduction)

**RIMS (Revenue Insurance Management System)** คือระบบที่พัฒนาขึ้นเพื่อเพิ่มประสิทธิภาพในการบริหารจัดการข้อมูลการจัดเก็บรายได้และลูกหนี้ค่ารักษาพยาบาลของโรงพยาบาล โดยเน้นการเชื่อมโยงข้อมูลจากระบบ HIS (HOSxP) เพื่อมาทำการตรวจสอบ, แยกประเภทลูกหนี้ตามผังบัญชีมาตรฐาน และเตรียมความพร้อมในการส่งเบิก (Claim) ไปยังกองทุนต่างๆ

### วัตถุประสงค์ (Objectives)
*   **Accuracy**: เพื่อให้การแยกประเภทลูกหนี้มีความถูกต้องตามรหัสบัญชีมาตรฐาน 1102050101.xxx และ 1102050102.xxx
*   **Efficiency**: เพื่อลดระยะเวลาในการตรวจสอบและนำเข้าข้อมูล STM (Statement) จากกองทุน (UCS, OFC, LGO, SSS)
*   **Transparency**: เพื่อให้ผู้บริหารและเจ้าหน้าที่การเงินสามารถติดตามสถานะการจ่ายเงิน (Eclaim Status) และยอดค้างรับได้แบบ Real-time
*   **Integration**: รวบรวมฟีเจอร์การเคลมสิทธิพิเศษต่างๆ (MIS Modules) ไว้ในที่เดียว

### ฟีเจอร์หลัก (Key Features)
1.  **Dashboard**: สรุปยอดลูกหนี้แยกตามสิทธิ และสถานะการตอบรับจาก STM
2.  **STM Import System**: นำเข้าไฟล์ตอบรับจากกองทุนเพื่อปรับปรุงยอดลูกหนี้อัตโนมัติ
3.  **Debt Management**: ตรวจสอบและยืนยันลูกหนี้รายตัว (Indiv) พร้อมรายงาน PDF/Excel
4.  **Claim Preparation**: เครื่องมือช่วยเตรียมข้อมูลส่งเบิกสำหรับสิทธิ UCS, OFC, LGO, BKK, SSS และอื่นๆ
5.  **MIS & Reports**: รายงานสถิติตามกลุ่มโรคสำคัญ (Sepsis, Stroke, STEMI) และรายงานการบริการ

## 2. การติดตั้ง (Installation Guide สำหรับ Developer)

### Pre-requisites (สิ่งที่ต้องเตรียม)
*   **PHP**: เวอร์ชัน 8.2 ขึ้นไป (โปรเจกต์ใช้ Laravel 12)
*   **Composer**: เวอร์ชันล่าสุด สำหรับจัดการ PHP Dependencies
*   **Node.js & NPM**: เวอร์ชัน 20.x ขึ้นไป (สำหรับ Vite)
*   **Database**: MySQL 8.0+ หรือ MariaDB 10.4+
*   **Version Control**: Git

### ขั้นตอนการติดตั้งบน Apache (AlmaLinux 9)
1.  **การ Clone Project**:
    ```bash
    cd /var/www/html
    git clone https://github.com/xmodify/h-rims.git rims
    cd rims
    ```
    *คำอธิบาย: ดาวน์โหลดซอร์สโค้ดจาก Repository และเข้าไปที่โฟลเดอร์โปรเจกต์*

2.  **การติดตั้ง Library (PHP & JS)**:
    ```bash
    composer install --no-dev --optimize-autoloader
    npm install
    npm run build
    ```
    *คำอธิบาย: ติดตั้ง Dependencies ของระบบ และทำการ Build ไฟล์ assets สำหรับการใช้งานจริง*

3.  **การตั้งค่า Environment**:
    ```bash
    cp .env.example .env
    nano .env
    ```
    *คำอธิบาย: คัดลอกไฟล์ .env.example เพื่อสร้างไฟล์ .env และแก้ไขข้อมูลการเชื่อมต่อ Database (DB_DATABASE, DB_USERNAME, DB_PASSWORD)*

4.  **การสร้าง App Key**:
    ```bash
    php artisan key:generate
    ```
    *คำอธิบาย: สร้างกุญแจเข้ารหัสสำหรับเซสชันและข้อมูลที่ต้องการความปลอดภัย*

5.  **การตั้งค่าสิทธิ์ (Permissions)**:
    ```bash
    chown -R apache:apache storage bootstrap/cache
    chmod -R 775 storage bootstrap/cache
    ```

### ขั้นตอนการติดตั้งบน XAMPP (Windows)
1.  เปิด **XAMPP Control Panel** และ Start Apache/MySQL
2.  ใช้ Terminal/CMD เข้าไปที่ `C:\xampp\htdocs\rims`
3.  รันคำสั่ง **Clone, Composer Install, NPM Install** และ **Key Generate** (เช่นเดียวกับด้านบน)
4.  **Config Virtual Host** (แนะนำ):
    *   แก้ไขไฟล์ `C:\xampp\apache\conf\extra\httpd-vhosts.conf`
    *   ตั้งค่า DocumentRoot ไปที่โฟลเดอร์ `public`
    ```apache
    <VirtualHost *:80>
        DocumentRoot "C:/xampp/htdocs/rims/public"
        ServerName rims.local
    </VirtualHost>
    ```

## 3. การใช้งาน (Usage Guide)

### [คำอธิบายภาพ: หน้าจอ Login]
*   **Path**: `/login`
*   ใช้สำหรับยืนยันตัวตนเข้าสู่ระบบ (ในเบื้องต้นจะใช้สิทธิ์ Admin หรือ Staff ที่ตั้งค่าไว้ในฐานข้อมูล)

### 3.1 หน้าสรุปผล (Dashboard)
*   **รายละเอียด**: แสดงภาพรวมรายได้แยกตามสิทธิ (OPD OFC, OPD UCS, IPD Homeward ฯลฯ)
*   **การใช้งาน**: ผู้ใช้สามารถดูตัวเลขยอดรวมและแนวโน้มการเคลมได้ทันที

### 3.2 ระบบนำเข้าข้อมูล (Import System)
*   **รายละเอียด**: ใช้สำหรับนำไฟล์ STM (Statement) จากกองทุนเข้าสู่ระบบ
*   **ขั้นตอนการใช้งาน**:
    1.  ไปที่เมนู **Import** บนแถบเมนูด้านซ้าย
    2.  เลือกประเภทที่ต้องการนำเข้า เช่น **STM UCS**
    3.  กดเลือกไฟล์ (รองรับ .zip หรือ .xlsx)
    4.  กดปุ่ม **Import/Save** เพื่อประมวลผล
    *   **[คำอธิบายภาพ: หน้าจอ Import STM UCS]**

### 3.3 ระบบจัดการลูกหนี้ (Debtor)
*   **รายละเอียด**: โมดูลหลักในการตรวจสอบความถูกต้องของลูกหนี้แยกตามรหัสผังบัญชี (เช่น 1102050101_xxx)
*   **ขั้นตอนการใช้งาน**:
    1.  เลือกหมวดลูกหนี้ที่ต้องการตรวจสอบ (เช่น **1102050101_103**)
    2.  เลือกช่วงวันที่ดึงข้อมูล (หรือวันที่รับบริการ)
    3.  กด **Search** เพื่อแสดงรายการ
    4.  ตรวจสอบยอดเงิน Income vs Claim
    5.  กดปุ่ม **Confirm** เพื่อยืนยันสถานะลูกหนี้ หรือกด **Export Excel** เพื่อส่งออกผลลัพธ์
    *   **[คำอธิบายภาพ: หน้าจอรายการลูกหนี้รายหมวดบัญชี]**

### 3.4 ระบบงานเคลม (Claim Management)
*   **รายละเอียด**: เตรียมความพร้อมข้อมูลสำหรับการส่งเบิก
*   **ขั้นตอนการใช้งาน**:
    1.  เลือกกองทุนเป้าหมาย (เช่น **Claim OP > UCS InCup**)
    2.  ระบบจะดึงรายการ Visit ที่ตรงตามเงื่อนไข
    3.  ผู้ใช้สามารถตรวจสอบความถูกต้องของสิทธิสวัสดิการเบื้องต้น
    4.  กดปุ่ม **Check/Lock** เพื่อล็อกรายการเตรียมส่งเบิก
    *   **[คำอธิบายภาพ: หน้าจอตรวจสอบข้อมูลก่อนส่งเบิก]**

### 3.5 รายงานสถิติและ MIS (Reports)
*   **รายละเอียด**: รายงานกลุ่มโรคและตัวชี้วัด เช่น Sepsis, Stroke, STEMI
*   **ขั้นตอนการใช้งาน**: เลือกรายงานที่ต้องการ > เลือกช่วงปีงบประมาณ > ระบบจะแสดงตารางสรุป
*   **[คำอธิบายภาพ: ตัวอย่างรายงาน PDF/Excel จากระบบ]**

## 4. การจัดการระบบ (Admin Settings)

*   **จัดการผู้ใช้งาน**: เพิ่ม/ลบ สิทธิ์เจ้าหน้าที่ในระบบ
*   **ตั้งค่าหลัก (Main Setting)**: กำหนดค่ามาตรฐานในการเชื่อมต่อ API และรหัสสถานบริการ
*   **ปีงบประมาณ**: ตั้งค่าช่วงเวลาเริ่มต้น-สิ้นสุดปีงบประมาณเพื่อการออกรายงานที่ถูกต้อง
